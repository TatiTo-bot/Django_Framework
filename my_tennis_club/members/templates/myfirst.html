Definición

Django es un framework de desarrollo web de código abierto para Python. Es una de las herramientas más populares para el desarrollo de aplicaciones web en Python y sigue el patrón de diseño MVT(Modelo Vista Plantilla)

Historia de Django

Django fue inventado por Lawrence Journal-World en 2003, para cumplir con los plazos cortos del periódico y al mismo tiempo satisfacer las demandas de los desarrolladores web experimentados, el lanzamiento inicial al público fue en julio de 2005.

Función de Django

Permite desarrollar aplicaciones web de manera rápida y eficiente.

Estructura de un proyecto Django

Un proyecto Django se compone de aplicaciones, que son módulos independientes que pueden ser reutilizados en diferentes proyectos. Cada aplicación tiene su propia estructura de directorios y archivos, la estructura se basa en elmodelo, vista y plantilla.

  • Modelo: Define la estructura de los datos y cómo se almacenan en la base de datos, los datos que desea presentar, gemeralmemte datos de una base de datos.
  • Vista: Contiene la lógica de negocio y maneja las solicitudes del usuario, las vistas son un controlador de solicitudes que devuelve la plantilla y el contenido relevante, según la solicitud del usuario.
  • Plantilla: Define cómo se presentan los datos al usuario, esta plantilla es un archivo de texto(como un archivo HTML) que contiene el diseño de la página web, con lógica sobre cómo mostrar los datos.

Ventajas y Desventajas de Django

Ventajas Desventajas
Desarrollo rápido: Agiliza el proceso de desarrollo web. Curva de aprendizaje: Puede ser pronunciada para principiantes.
Seguridad: Protección contra vulnerabilidades comunes. Menor flexibilidad: Menos adaptable para personalizaciones extremas.
Panel de administración: Gestión de datos fácil e intuitiva. ORM restrictivo: Puede limitar consultas SQL personalizadas.
Funcionalidades integradas: Muchas herramientas listas para usar. Rendimiento en tiempo real: No ideal para aplicaciones en tiempo real.
Comunidad y documentación: Gran apoyo y recursos disponibles. Complejidad en APIs REST: Puede ser complicado para construir APIs puras.
Escalabilidad: Diseñado para aplicaciones a gran escala.
Patrón MVT: Facilita el desarrollo y mantenimiento del código.
ORM: Simplifica las interacciones con la base de datos.
